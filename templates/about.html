FM Monitor

Outil de supervision de diffusion radio FM conçu pour les radios, régies techniques et prestataires broadcast. Surveille en continu la présence et la qualité du signal FM via un dongle RTL-SDR, envoie des alertes email en cas de coupure, décode les données RDS et offre un tableau de bord accessible depuis n'importe quel navigateur.

Open Source Raspberry Pi / Linux RTL-SDR Python / Flask
Comment fonctionne FM Monitor
📡
1. Capture RF
Le dongle RTL-SDR reçoit la porteuse FM. rtl_fm démodule le signal en audio PCM brut.
🔊
2. Analyse audio
FM Monitor calcule le niveau RMS en dBFS en temps réel pour détecter silences et modulation.
📻
3. Décodage RDS
redsea extrait les données RDS du flux MPX : PS, RadioText, PI Code.
🌐
4. Dashboard
Flask expose un dashboard web avec alertes email, historique SQLite et streaming Icecast2.
La diffusion FM

📡 La porteuse RF

La diffusion FM utilise une porteuse radio dans la bande 87.5 – 108 MHz. L'information audio est encodée par modulation de fréquence : la fréquence de la porteuse varie proportionnellement à l'amplitude du signal audio. La déviation maximale est de ±75 kHz, ce qui donne à la FM sa largeur de canal de ~200 kHz.

〰️ Le signal MPX (Multiplex)

Ce qui module la porteuse est un signal composite MPX, composé de plusieurs sous-porteuses :

  • 0–15 kHz : audio mono (L+R)
  • 19 kHz : pilote stéréo
  • 23–53 kHz : signal stéréo (L–R) DSB-SC
  • 57 kHz : sous-porteuse RDS
Le RDS — Radio Data System

Norme IEC 62106 permettant aux stations FM de transmettre des données numériques sur la sous-porteuse à 57 kHz. Débit : 1187,5 bits/s, organisé en groupes de 104 bits.

PS — Programme Service
Nom de la station (max 8 caractères). Groupe 0A/0B. Reçu en ~1-2 s.
GRAFFITI
RT — RadioText
Texte libre jusqu'à 64 caractères. Titre/artiste en cours. Groupe 2A/2B. ~20 s pour une trame complète.
ERIK TRUFFAZ - Le compleme
PI — Programme Identification
Code hexadécimal unique 16 bits identifiant la station. Permet le suivi réseau.
0xFA41
Pourquoi le RadioText peut être partiel ?

Le RT est découpé en segments de 4 caractères transmis successivement. Si la réception démarre en cours de séquence, ou si l'encodeur RDS ne génère pas de trame conforme, seul un RT partiel est disponible. FM Monitor attend une séquence stable avant d'afficher le RadioText.

rds-station-db

FM Monitor utilise rds-station-db, une base de données de stations FM maintenue sur GitHub, pour identifier automatiquement la station captée et récupérer son logo à partir du code PI RDS.

Comment FM Monitor identifie la station
  1. Réception du code PI RDS (ex: FA41)
  2. Recherche dans rds-station-db par code PI
  3. Récupération du nom et du logo associé
  4. Logo mis en cache — rechargement toutes les 24h

Installation RTL-SDR Blog V4 (Raspberry Pi / Debian / Ubuntu) 
# 1. Bloquer le driver noyau générique
echo 'blacklist dvb_usb_rtl28xxu' | sudo tee /etc/modprobe.d/blacklist-rtl.conf
sudo modprobe -r dvb_usb_rtl28xxu 2>/dev/null

# 2. Dépendances
sudo apt update && sudo apt install -y git cmake libusb-1.0-0-dev build-essential pkg-config

# 3. Compiler le fork officiel RTL-SDR Blog
git clone https://github.com/rtlsdrblog/rtl-sdr-blog
cd rtl-sdr-blog && mkdir build && cd build
cmake .. -DINSTALL_UDEV_RULES=ON
make -j$(nproc) && sudo make install && sudo ldconfig

# 4. Règles udev (accès sans sudo)
sudo cp ~/rtl-sdr-blog/rtl-sdr.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger

# 5. Test
rtl_test -t
⚠️ Bias-T (V4) : Le RTL-SDR Blog V4 dispose d'une alimentation antenne (Bias-T) de 4,5 V désactivée par défaut. Ne l'activez que si votre antenne ou amplificateur le nécessite : rtl_biast -b 1
Analyse du signal MPX

Le signal FM broadcast n'est pas un simple signal audio — c'est un multiplex (MPX) complexe transportant plusieurs informations en parallèle sur différentes sous-porteuses.

Déviation FM

0–75 kHz

Amplitude de la variation instantanée de la fréquence de la porteuse. Maximum autorisé : ±75 kHz. Une sur-déviation dégrade la qualité et peut empiéter sur les canaux adjacents.

0–60 kHz ✓ 60–70 kHz ⚠ >70 kHz ✗

Ton pilote 19 kHz

19 000 Hz

Sinusoïde pure à 19 kHz signalant la présence d'un programme stéréo. Le récepteur l'utilise comme référence de phase pour régénérer la sous-porteuse à 38 kHz. Son absence indique une diffusion mono. Niveau : 8 à 10 % de la déviation totale.

Signal stéréo L−R (38 kHz)

23–53 kHz

Transmis en DSB-SC centré à 38 kHz. Contient la différence Gauche−Droite. En combinant le signal mono (L+R) avec ce signal, on reconstitue L et R. La porteuse 38 kHz n'est pas transmise — elle est régénérée depuis le pilote 19 kHz.

Sous-porteuse RDS (57 kHz)

57 000 Hz

Le RDS (CEI 62106) transmet des données à 1 187,5 bit/s sur une sous-porteuse à 57 kHz — le triple du pilote, en quadrature de phase. Transporte PS, RT, PI, PTY, CT et AF. Niveau typique : 2 à 5 % de la déviation totale.

Puissance MPX

dBFS

Niveau RMS global du signal multiplex après démodulation FM, en dBFS. Représente l'énergie totale combinée : audio L+R, stéréo L−R, pilote et RDS. Indicateur global de présence et vigueur du signal.

SNR — Rapport signal/bruit

dB

Rapport entre le niveau du signal audio utile (100 Hz–15 kHz) et le plancher de bruit (60–75 kHz). Un SNR faible trahit une réception dégradée ou une interférence.

>40 dB ✓ 25–40 dB ⚠ <25 dB ✗
