FM Monitor

FM Monitor est un outil de supervision de diffusion radio FM conçu pour les radios, régies techniques et prestataires broadcast. Il surveille en continu la présence et la qualité du signal FM capté via un dongle RTL-SDR, envoie des alertes email en cas de coupure, décode les données RDS et offre un tableau de bord accessible depuis n'importe quel navigateur.

Open Source Raspberry Pi / Linux RTL-SDR Python / Flask

Comment fonctionne FM Monitor

📡
1. Capture RF
Le dongle RTL-SDR reçoit la porteuse FM. rtl_fm démodule le signal FM en audio brut PCM.
🔊
2. Analyse audio
FM Monitor calcule le niveau RMS en dBFS en temps réel pour détecter les silences et mesurer la modulation.
📻
3. Décodage RDS
redsea extrait les données RDS du flux MPX : PS, RadioText, PI Code.
🌐
4. Dashboard
Flask expose un dashboard web avec alertes email, historique BDD SQLite et streaming audio Icecast2.

La diffusion FM

📡 La porteuse RF

La diffusion FM utilise une porteuse radio dans la bande 87.5 – 108 MHz. L'information audio est encodée par modulation de fréquence (FM) : la fréquence de la porteuse varie proportionnellement à l'amplitude du signal audio. La déviation maximale est de ±75 kHz, ce qui donne à la FM sa largeur de canal de ~200 kHz.

〰️ Le signal MPX (Multiplex)

Ce qui module la porteuse n'est pas directement l'audio stéréo, mais un signal composite appelé MPX (multiplex), composé de plusieurs sous-porteuses :

  • 0 – 15 kHz : audio mono (L+R) — compatibilité mono
  • 19 kHz : sous-porteuse pilote stéréo — indique aux récepteurs que le signal est stéréo
  • 23 – 53 kHz : signal stéréo (L–R) en DSB-SC autour de 38 kHz
  • 57 kHz : sous-porteuse RDS (Radio Data System)

Le RDS — Radio Data System

Le RDS est une norme (IEC 62106) permettant aux stations FM de transmettre des données numériques en parallèle de l'audio, modulées sur la sous-porteuse à 57 kHz. Le débit est de 1187.5 bits/s, organisé en groupes de 104 bits.

PS — Programme Service
Nom de la station affiché sur l'autoradio. Limité à 8 caractères. Transmis en groupe 0A/0B, se reçoit rapidement (~1-2 secondes).
GRAFFITI
RT — RadioText
Texte libre jusqu'à 64 caractères (RT+ : 128). Généralement le titre et l'artiste en cours. Transmis en groupe 2A/2B, une trame complète prend ~20 secondes.
ERIK TRUFFAZ - Le compleme
PI — Programme Identification
Code hexadécimal unique sur 16 bits identifiant la station. Permet la commutation réseau et le suivi de station même en changement de fréquence. Exemple : 0xFA41 = Graffiti Urban Radio.
0xFA41
Pourquoi le RadioText peut être partiel ?

Le RT est découpé en segments de 4 caractères, transmis successivement. redsea reçoit ces segments au fil de la réception et les reconstitue. Si la réception démarre en cours de séquence, ou si l'encodeur RDS de la station ne génère pas de trame "complète" conforme à la norme, seul un RT partiel est disponible. FM Monitor attend une séquence stable avant d'afficher le RadioText.

Radio Browser

radio-browser.info est une base de données collaborative et open source répertoriant des milliers de stations radio du monde entier. FM Monitor l'utilise pour récupérer automatiquement le logo de la station captée, en croisant le nom PS RDS avec la fréquence.

Comment FM Monitor trouve le logo
  1. Réception du PS RDS (ex: "GRAFFITI")
  2. Recherche sur Radio Browser avec le nom complet de la station (config) + fréquence
  3. Priorité aux résultats dont la fréquence correspond exactement
  4. Fallback sur le premier résultat avec un logo valide (PNG/JPG/SVG)
  5. Logo mis en cache — une seule requête par session

Dongles RTL-SDR supportés

Dongle Puce Drivers Notes
RTL-SDR Blog V3 RTL2832U + R820T2 rtl-sdr standard Drivers inclus dans la plupart des distributions. Plug & play.
RTL-SDR Blog V4 Recommandé RTL2832U + R828D rtl-sdr-blog (fork) Meilleure sensibilité FM, TCXO 1ppm. Nécessite le fork officiel et le blacklist du driver noyau.
Nooelec NESDR SMArt v5 RTL2832U + R820T2 rtl-sdr standard Compatible standard. Boîtier aluminium anti-interférences.

Installation RTL-SDR Blog V4 (Raspberry Pi / Debian / Ubuntu) 
# 1. Bloquer le driver noyau générique
echo 'blacklist dvb_usb_rtl28xxu' | sudo tee /etc/modprobe.d/blacklist-rtl.conf
sudo modprobe -r dvb_usb_rtl28xxu 2>/dev/null

# 2. Dépendances
sudo apt update
sudo apt install -y git cmake libusb-1.0-0-dev build-essential pkg-config

# 3. Compiler le fork officiel RTL-SDR Blog
git clone https://github.com/rtlsdrblog/rtl-sdr-blog
cd rtl-sdr-blog && mkdir build && cd build
cmake .. -DINSTALL_UDEV_RULES=ON
make -j$(nproc) && sudo make install && sudo ldconfig

# 4. Règles udev (accès sans sudo)
sudo cp ~/rtl-sdr-blog/rtl-sdr.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger

# 5. Test
rtl_test -t
⚠️ Bias-T (V4) : Le RTL-SDR Blog V4 dispose d'une alimentation antenne (Bias-T) de 4.5V désactivée par défaut. Ne l'activez que si votre antenne ou amplificateur le nécessite : rtl_biast -b 1
